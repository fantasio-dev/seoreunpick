# 서른픽 — 결정 히스토리 & 배경 (왜 이렇게 됐나)

> 현재 상태는 `CONTEXT.md`. 이 문서는 **"왜"** 와 **이미 논의/거절한 것** 을 담는다.
> 같은 결정을 다시 뒤집기 전에 여기부터 읽을 것.

## 1. 제품 방향 (가장 중요)
- **경쟁자는 카카오톡 일정 투표.** 모두 이미 카톡에 있어서, "더 나은 투표"로는 못 이긴다.
- **유일한 진짜 차별점 = 필수 참석자(앵커) 조건.** △(애매 상태), 미응답 추적, 과반 판단은 카톡도 대충 된다. 앵커만 카톡이 구조적으로 못 한다.
- **불편한 진실**: 똑똑한 추천의 수혜자는 주최자 1명, 비용(번거로움)은 참여자 전원이 낸다. 그래서 "더 좋아도 안 쓰이는" 함정이 있음.
- **결론 포지셔닝**: "투표 앱"이 아니라 **"모임을 끝까지 성사시키는 도구"** 로 가야 카톡이 못 따라온다. → 그래서 "카톡 복붙 한 줄"(구현됨) + 마감/자동확정(다음)이 핵심 방향.
- 타겟을 좁힌다면: **특정인이 빠지면 의미 없는 모임**(승진축하/환송회/상견례), 또는 **정족수 미달=금전손해인 모임**(풋살 구장 등)이 앵커+정족수 가치가 가장 큼.

## 2. 본인인증 / 카톡 연동 (논의 완료)
- 무인증(이름 선택)은 **의도적 트레이드오프**. 친구용이라 OK로 둠.
- 진짜 본인확인 필요 시 → **카카오 로그인**(이름 고르기 단계도 사라짐). 셋업 + 로그인 마찰 추가.
- 카톡 연동 2종: ① **카카오 공유 SDK**(복사 대신 리치 카드 전송, 쉬움, JS키만) ② 카카오 로그인(본인확인, 셋업 더). 지금은 OG로 충분하다고 판단.

## 3. 핵심 기술 결정 & 이유
- **위치**: 회사 영역 `~/woowa/src` → 개인 영역 `~/woowa/dev` 로 이동(토이프로젝트라).
- **DB: better-sqlite3 → @libsql/client(Turso)**. 이유: Vercel 서버리스는 영속 디스크가 없어 파일 SQLite 불가. libSQL 하나로 로컬 file: + 운영 Turso 를 같은 코드로 처리. (Node 25에서 둘 다 동작 확인함)
  - 모든 DB 함수 async 화, 호출부에 await. 시드는 `INSERT OR IGNORE` 로 서버리스 콜드스타트 race-safe.
- **디자인: 보라(#6257e6) → 토스블루(#3182F6) 전면 개편.** "구리다"는 피드백 → 토스/당근 톤, Pretendard.
- **투표 △ 제거 → O/X 2단계.** "최대한 미니멀" 선택의 결과. (△는 카톡 대비 차별점이긴 했지만 단순함 우선)
- **정족수 과반 자동(숨김)**, 방장 이름 필드 제거(첫 멤버=방장). 폼 단순화.
- **결과 페이지 미니멀화**: 추천 1개 크게 + 안 한 사람 + 다른 날짜(사유 칩). 사람×날짜 그리드 제거.

## 3-1. 마감일 + 자동 확정 (구현됨)
- **마감일은 날짜 단위(YYYY-MM-DD), 선택.** 시각 단위는 친구용엔 과함. 마감일 당일까지 투표 가능, **다음 날부터 마감**(KST 기준, `todayKstYmd() > deadline`). 서버가 Vercel UTC라 항상 KST로 환산.
- **자동 확정 트리거 = "마감"으로 통일.** TODO엔 "정족수 도달/마감 시"였지만 **정족수 도달 즉시 락은 의도적으로 안 함**(더 좋은 날이 나올 수 있는데 놀랍게 잠겨버림). 대신 마감 시점에 **정족수+앵커를 충족한(green) 대표 날짜**를 확정 → "정족수"는 *무엇을 확정할지 고르는 기준*으로만 사용.
- **green 이 없으면 자동 확정 안 함.** 아무도 동의 안 한 날을 강제로 잡지 않는다. 투표만 닫고 방장이 직접 고르도록 결과에 안내.
- **cron 없음 → lazy 자동 확정.** 별도 스케줄러/서버리스 크론 없이 `getPollBundle` 읽는 시점에 1회 처리(이후 status=confirmed라 스킵). OG/메타/결과 어느 경로로 들어와도 일관되게 확정 상태가 보임.
- **마감 후 투표 차단**은 서버 액션(`submitVoteAction`)에서도 막아 UI 우회 방지.

## 3-2. 방장 권한 + 방 수정/삭제 (구현됨)
- **카카오 로그인 대신 "관리 링크(비밀 토큰)" 채택.** 로그인은 마찰이 커서 "친구용 무인증" 결정(2장)과 충돌 → 인증 없이 권한을 주려고 `manage_token` 발급. 방 생성자는 `?t=토큰` 링크를 받고, 그 링크 보유자만 확정/수정/삭제.
- **토큰은 서버에서만 검증(`verifyManage`), 클라이언트에 노출 금지.** PollMeta/PollBundle 에 넣지 않음. 권한 판단(`canManage`)만 페이지에서 하고, 버튼은 방장에게만 렌더 + 서버 액션에서도 토큰 재검증(UI 우회 방지).
- **레거시 방(토큰 null)은 개방 유지.** 기존에 만든 방까지 잠그면 못 쓰게 되므로 하위호환으로 누구나 가능.
- **수정 범위 = 제목/정족수/마감일 + 멤버.** 날짜 편집만 제외(기존 투표와의 정합성 처리가 커서 — 필요하면 방 새로 만들기).
  - **멤버 편집 정합성**: 멤버는 **id 기준**으로 동기화(`updateMembers`). 이름변경/앵커토글은 그 멤버의 **투표 보존**(id 유지), **삭제**한 멤버는 **그 투표도 함께 삭제**, 새 멤버는 추가(투표 없음=미응답). 멤버 수가 줄면 **정족수 자동 클램프**.
- 한계: 토큰 링크를 그대로 공유하면 유출됨(친구용이라 감수). 공유 버튼들은 토큰 없는 URL만 생성.

## 3-3. 미응답 콕 독촉 / 확정 축하 (구현됨)
- "콕"은 **개인 맞춤 문구 복사**(이름 호명)로 구현. 자동 푸시/알림은 인증·연동이 없어 불가 → 복사 후 사람이 보내는 방식 유지(기존 카톡 복붙 철학과 동일).
- 축하 연출은 라이브러리 없이 CSS 컨페티. 매번 뜨면 거슬려서 **`sessionStorage` 로 방당 1회**만.

## 3-5. "내가 만든 방" 목록 (구현됨)
- **무인증이라 서버는 "누가 만든 방"인지 모름** → 계정/로그인 없이 목록을 주려고 **이 기기 localStorage** 에 기록(`lib/myRooms.ts`). 방장이 결과 페이지를 `?t=토큰` 으로 열면(`RememberRoom`) id/제목/토큰 저장, 홈에서 `MyRooms` 가 노출.
- **결과 페이지 진입 시 기록(생성 시점이 아니라).** 생성 직후 `?t=` 결과로 리다이렉트되므로 자동 저장되고, 나중에 관리 링크로 다시 들어와도 목록에 복원됨.
- 한계: 기기/브라우저 로컬이라 **기기 바꾸면 안 보임**(관리 링크 자체가 진짜 권한). 서버 목록은 인증 도입 전엔 안 함. 토큰을 localStorage 에 두지만 클라 전용이고 이미 URL 로 들고 다니는 값이라 추가 노출 아님.

## 3-4. 카카오 공유 SDK (구현됨, 키는 운영자 몫)
- **공유 SDK만 채택(로그인 X).** JS키만으로 리치 카드 전송 가능, 사용자 터치는 오히려 감소(복사→앱전환→붙여넣기 → 탭 1회).
- **키 없으면 자동 폴백.** `NEXT_PUBLIC_KAKAO_JS_KEY` 미설정 시 SDK 미로딩 + 웹공유/복사로 폴백 → 키 없는 환경에서도 안전. 비밀값 아님(공개 JS키)이라 env 로만, 코드/깃 금지 원칙은 동일.

## 4. 시도했다 버린 것 (다시 하지 말 것)
- **폰에서 보기**: 로컬 LAN IP(172.x) → 폰이 LTE라 실패. → **ngrok** → 회사망 TLS 검사(MITM)로 cert 실패(정책 차단 가능성). → **cloudflared 퀵터널(QUIC)** 로 성공(임시). → 최종은 **Vercel 배포**가 정답.
  - 교훈: **회사망에선 CLI 툴(ngrok/turso/vercel)이 막힐 수 있다. 브라우저 대시보드로 우회.** cloudflared(QUIC)는 통과.
- **OG 폰트 로딩**: `fetch(new URL('./font.otf', import.meta.url))` → webpack이 상대경로(`/_next/static/...`)로 바꿔 Node fetch가 절대 URL 아니라 실패. → **CDN 절대 URL fetch**(`lib/og-font.ts`)로 해결. 로컬 폰트 파일은 제거함.
- **OG 이모지**: satori가 이모지 폰트 없으면 미렌더 → OG 이미지에선 이모지 빼고 텍스트만.
- **결과 세그먼트 OG**: 부모 `poll/[id]/opengraph-image` 를 자식 `result/` 가 자동 상속 안 함 → `result/opengraph-image.tsx` 에서 재export.

## 5. 미해결 / 주의
- **카톡 OG 캐시**: 전에 공유한 링크는 미리보기 안 바뀜. 새 링크/`?v=1` 우회.
- ~~권한 없음~~ → 해결: `manage_token` 으로 방장만 확정/수정/삭제(3-2).
- ~~방 수정/삭제 불가~~ → 해결: manage 페이지(제목/정족수/마감일/멤버 + 삭제). 날짜 편집만 여전히 불가.
- **방장 토큰 유출 가능성**: 토큰 링크를 그대로 공유하면 권한 노출(친구용이라 감수).
- **카카오 공유는 키 필요**: `NEXT_PUBLIC_KAKAO_JS_KEY` 없으면 폴백만. 키 발급 + 카카오 콘솔 도메인 등록은 운영자 작업.
- **자동 확정 lazy 한계**: 마감 후 아무도 안 열면 확정 안 찍힘 → 필요 시 Vercel Cron 도입.
- 2FA: Vercel 계정 보안 스킵함. 나중에 켜기 권장.
